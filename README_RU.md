Добавлена полноценная светлая тема и переключатель в меню аккаунта. Выбор сохраняется на устройстве. Остальная логика v6.0.4 сохранена.

# BOONWAVE Clean Workspace v6.0.0

Новая тестовая PWA-сборка рабочего пространства, написанная с чистой архитектурой. Старый рабочий стол v5.4.5 не используется.

## Что реализовано

- сохранены утверждённые экран загрузки и форма входа;
- чистая верхняя панель: знак, «Личное / Проекты», меню;
- свободное рабочее поле с pan и pinch-to-zoom;
- двойное нажатие по пустому полю — вписать все карточки;
- одинарное нажатие — выбрать карточку;
- быстрое двойное нажатие — открыть просмотр;
- удерживание карточки 2 секунды — открыть редактирование;
- перемещение карточки одним пальцем;
- масштаб карточки двумя пальцами с плавным откликом и фиксацией на трёх уровнях;
- компактный, рабочий и расширенный вид карточек;
- карточки: проект, рабочий процесс, человек, идея, цель;
- отдельная карточка рабочего процесса с этапами, задачами, людьми и затратами;
- ключевые материалы проекта остаются в основной карточке;
- горизонтальная микро-галерея изображений, PDF, SVG и производственных файлов;
- полноэкранный просмотр вложений;
- хранение бинарных вложений в IndexedDB;
- волновые связи с автоматическим выбором стороны карточки;
- разделы «Сегодня», «Результаты», «Архив»;
- локальные аккаунты, экспорт и импорт JSON;
- офлайн-кэш и автоматическое обновление service worker.

## Визуальное знакомство

Кнопка «Визуальное знакомство» открывает демонстрационное рабочее пространство с проектом, рабочим процессом, человеком и идеей. Обычная регистрация создаёт пустой аккаунт.

## Публикация на GitHub Pages

1. Создайте новый репозиторий или отдельную ветку.
2. Загрузите **содержимое этой папки** в корень репозитория.
3. Откройте `Settings → Pages`.
4. В разделе `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку и папку `/ (root)`.
6. После публикации откройте ссылку в Safari.

Для проверки без GitHub:

```bash
python3 -m http.server 8080
```

Откройте `http://localhost:8080`.

## Ограничения тестовой версии

- аккаунты и данные локальные, без облачной синхронизации;
- PDF-просмотр зависит от встроенного просмотрщика браузера;
- JSON-экспорт содержит структуру и метаданные, но не включает бинарные файлы из IndexedDB;
- совместная работа пользователей пока не подключена.
