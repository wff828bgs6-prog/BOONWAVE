# Матрица проверки service-команд

Проверяется:

- запись в storage выполняется до изменения store;
- ошибка storage не меняет состояние приложения;
- выбирается только активный этап;
- задача создаётся в выбранном этапе;
- перенос между этапами сохраняет вложения;
- изменение порядка не затрагивает другие этапы;
- архивирование выбранного этапа безопасно переключает selection;
- архивирование текущей задачи снимает отметку «Я сейчас»;
- восстановление возвращает объект в active lifecycle.
