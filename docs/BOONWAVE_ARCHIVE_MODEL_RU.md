# BOONWAVE — модель архива

Дата фиксации: 2026-07-04
Статус: проектная модель перед реализацией.

## Зачем нужен архив

Архив нужен, чтобы пользователь мог убрать карточку с рабочего стола без риска потерять данные.

Это безопаснее, чем удаление. В качественном приложении важная информация не должна исчезать одним нажатием.

## Простая логика

Карточка может быть в одном из состояний:

1. Активная.
2. В архиве.
3. Убрана окончательно.

Активная карточка видна на рабочем столе.
Карточка в архиве не мешает на рабочем столе, но её можно открыть, посмотреть и восстановить.
Окончательное удаление доступно только из архива и только после подтверждения.

## Что делает команда «Архивировать»

Команда «Архивировать» должна:

- проверить, что выбрана карточка;
- не трогать файлы и материалы карточки;
- не удалять связи;
- поменять состояние карточки на «в архиве»;
- скрыть карточку с рабочего стола;
- снять выделение с этой карточки;
- показать понятное сообщение пользователю.

## Что делает раздел «Архив»

Раздел «Архив» должен показывать список архивированных карточек.

Для каждой карточки в архиве нужно видеть:

- тип карточки;
- название;
- краткое описание;
- дату архивации;
- кнопку открыть;
- кнопку восстановить;
- кнопку убрать окончательно.

## Восстановление

Команда «Восстановить» должна вернуть карточку из архива на рабочий стол.

После восстановления карточка должна появиться на прежнем месте, если это возможно.

## Окончательное удаление

Окончательное удаление разрешено только внутри архива.

Перед действием нужно отдельное подтверждение.

При окончательном удалении нужно решить:

- что делать со связями;
- что делать с файлами;
- что делать с назначениями контактов;
- что делать с задачами и этапами, если они связаны с карточкой.

На первом этапе окончательное удаление лучше оставить заблокированным, пока не будет описана полная каскадная логика.

## Связи

При архивации связи не удаляются.

На рабочем столе связи архивированной карточки не показываются, потому что сама карточка скрыта.

После восстановления связи снова могут отображаться.

## Файлы и материалы

При архивации файлы не удаляются.

Удаление файлов возможно только при окончательном удалении и только после проверки, используются ли эти файлы ещё где-то.

## Системные ограничения

Карточки старого типа «Я Есмь» не должны использоваться как основа новой логики.

Контакты должны быть отдельным разделом, а не обычной карточкой «Человек» на рабочем столе.

## Как это будет выглядеть на экране

В окне «Дополнительные действия» есть кнопка «Архивировать».

После нажатия карточка исчезает с рабочего стола, но не теряется.

В этом же окне есть кнопка «Архив». Она открывает список архивированных карточек.

Внутри архива пользователь может посмотреть карточку и восстановить её.

## Почему это правильно для App Store

Это снижает риск случайной потери данных.

Это делает приложение предсказуемым.

Это отделяет безопасное скрытие карточки от опасного окончательного удаления.

Это позволяет позже добавить историю действий и резервные копии без переделки всей логики.

## Следующий шаг реализации

1. Добавить состояние карточки: active или archived.
2. Научить рабочий стол не показывать архивированные карточки.
3. Сделать команду «Архивировать».
4. Сделать список архива.
5. Сделать восстановление.
6. Только после этого проектировать окончательное удаление.
